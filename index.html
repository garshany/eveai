01

Capabilities

TG
grammY · discord.js

Telegram & Discord

The same agent runtime answers in a Telegram private chat or a Discord DM. Slash commands, character switching, and chunked replies on both.

SSO
OAuth2 · JWKS

EVE SSO linking

Link characters through EVE Single Sign-On. Tokens are encrypted at rest, auto-refreshed, and access is scope-aware — gated per character and chat.

ESI
live · cached

Live ESI data

Character, corp, market, assets, skills, industry, mail, location — fetched live from ESI with ETag revalidation and CCP-honoring cache timers.

KB
real-time

Killboard intel

zKillboard & EVE-KILL: real-time kill alerts, PvP stats, battle reports, observed fits, and gate-camp signals streamed straight to your chat.

RT
danger scan

Route planning

Danger-aware routing with live kill density, gate-camp detection, Thera/Turnur shortcuts, and a monitor mode that watches the road while you fly.

OS
inference

OSINT & SDE

Residence/staging inference from kill activity, plus fast local SDE lookups for items, systems, dogma, blueprints and routes — no ESI round-trip.

02

What updates automatically

Everything live from the EVE API refreshes on its own — killfeeds, character data, tokens, even the API schema. Only the static game data (SDE) is a deliberate manual step, because it changes only on EVE patches.

03

Architecture

One Node.js process. SQLite only. No queues, workers, webhooks, or web frontend — just the bots, the agent loop, and the EVE integrations.

Telegram private chat ─┐
                       ├─▶  shared agent runtime  ─▶  OpenAI Responses API
Discord DM ────────────┘            │
                                          ▼
                    hosted tool_search + deferred ESI / SDE / killboard tools
                                          │
                                          ▼
                              SQLite  +  local SDE  +  EVE SSO

Browser ─▶ Fastify ─▶ EVE SSO callback + /health ─▶ same SQLite state
04

Self-host it

$ terminal — first launch
# clone & install
git clone https://github.com/garshany/eveonline_ai eveai
cd eveai && cp .env.example .env   # add your tokens
npm install

# download + load EVE static data (SDE)
npm run setup

# fly
npm run dev
  • Node.js 20+ and npm
  • At least one bot token — Telegram (@BotFather) and/or Discord
  • An EVE developer app for SSO & ESI
  • An OpenAI API key (official Responses API)
  • No Redis, Postgres, queues, workers, webhooks, or frontend
