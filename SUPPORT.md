# Support

## Questions

Use GitHub Discussions when enabled. If Discussions are not enabled, open a regular issue with a clear question and sanitized environment details.

## Bugs

Use the bug report template and include:

- Node.js version
- OS/deployment type
- relevant env names, not secret values
- reproduction steps
- sanitized logs

Do not paste `.env`, database files, Telegram or Discord tokens, OpenAI API keys, EVE SSO secrets, server IPs, SSH usernames, or private runbooks.

## Security

Do not open public issues for vulnerabilities involving tokens, sessions, private ESI data, auth bypass, or account takeover paths. Follow [SECURITY.md](./SECURITY.md).

## EVE Online Data

For gameplay/data issues, clarify whether the source is local SDE, ESI, EVE-KILL, EVE-Scout, or web search. This helps maintainers reproduce the exact data path.
